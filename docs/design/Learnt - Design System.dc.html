The Learnt design language

A dark-first, diagrammatic system tuned for one unmistakable active task at a time. Restrained borders over heavy shadow, one signal accent for active state, a separate accent for retry — never color alone to carry meaning.

This page documents tokens, components and their states, recoverable edge states, the architecture-to-design map, and the build handoff. Every value below is the literal value used in the prototype.

Spacing, radii & elevation

SPACING · 4px base
4 · 8 · 12 · 16 · 24 · 32 · 48
RADII
6 chips · 9 controls · 14 cards
ELEVATION

Depth comes from borders (0.07–0.12 white) and surface steps, not shadow. The only "glow" is a 4–5px ring on the single active element.

ICON PRINCIPLES
  • Line icons, 1.4–1.6 stroke, 24-grid.
  • Diagrammatic motif: nodes + connecting paths.
  • Unfamiliar icons always paired with a label or tooltip.
  • No filled/duotone, no AI "sparkle".
MOTION PRINCIPLES
  • Purpose: spatial continuity on mode change, side-path, evaluation, return.
  • 150–280ms; ease; translate over opacity so content never traps invisible.
  • No looping/decorative animation. prefers-reduced-motion collapses all to ~0ms.

Implementation handoff

Page hierarchy
AppShell
├─ SubjectLibrary

├─ LearningWorkspace

│  ├─ ActivityFrame
│  │  ├─ ContentBlockRenderer

├─ ConceptMap (overlay)
└─ SessionRecap
Responsive rules
  • ≥1180px: rail + stage + context panel.
  • 761–1180: context panel auto-collapses; rail stays.
  • ≤760: single-task stage; rail + panel become drawers/sheets; map is a dedicated view; primary action sticky.
  • Flow/Test modes hide the context panel at any width.
Interaction rules
  • Thread: entering a related concept sets a side path; the anchor + unfinished response persist; Park stores it, Return restores the exact point. No modal on path change.
  • Modes: change density/support, never the layout skeleton — Zoom/Map open as overlays over a still-mounted workspace.
  • Evaluation: response preserved across Retry; Rescue appears after stalled attempts.
  • Keyboard: options 1–4 / A–D, Enter submits, Esc closes overlay → returns to thread, visible focus rings throughout.
Priorities & deferred
P0 — shell, library, workspace activity frame, 4 evaluation states, thread anchor.
P1 — concept map, modes, recap, extension frame + fallback.
P2 — mobile drawers, multi-head extension, large-graph clustering.
Possible future contract fields

Concept.relationKind (prerequisite | related), Activity.confidenceRequested, and Session.sidePaths[] are assumed by the design and not yet in the canonical contracts — propose as additions, do not assume silently.

Required assets: Hanken Grotesk + JetBrains Mono (Google Fonts), inline SVG icon set (24-grid line). No raster assets, no per-subject cover images — subject identity is a neutral mono glyph + tint.
